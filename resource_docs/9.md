## What is React?

React is a JavaScript library for building user interfaces. It uses a component-based architecture where each piece of UI is an isolated, reusable component.

## Core Concepts

### JSX

```jsx
const element = <h1 className="title">Hello, {name}!</h1>;
```

JSX is syntactic sugar for `React.createElement()`. It looks like HTML but compiles to JavaScript.

### Components

```jsx
// Function component
function Welcome({ name }) {
  return <h1>Hello, {name}</h1>;
}

// Usage
<Welcome name="Alice" />
```

### Props vs State

| Concept | Props | State |
|---------|-------|-------|
| Owned by | Parent component | Current component |
| Mutable? | No (read-only) | Yes (via setter) |
| Purpose | Pass data down | Manage internal data |

### Event Handling

```jsx
function Button() {
  const handleClick = () => alert("Clicked!");
  return <button onClick={handleClick}>Click Me</button>;
}
```

1. Think in components â€” break UI into small, reusable pieces.
2. Props flow down, events bubble up.
3. Never mutate state directly â€” always use the setter function.
